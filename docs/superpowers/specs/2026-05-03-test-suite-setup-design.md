# Automated Test Suite Setup Design

## Overview
Set up a standard, automated testing environment using `pytest` and `pytest-cov` to run existing tests in the `tests/` directory and generate code coverage reports for the `src/` directory.

## Requirements
- Python dependencies: `pytest`, `pytest-cov`.

## Configuration
A `pytest.ini` file will be created in the project root to standardize test execution.

**pytest.ini configuration:**
- `testpaths = tests`: Instructs pytest to only look in the `tests` directory for tests.
- `addopts`: Default command-line options.
  - `--cov=src`: Measures coverage for the `src` directory.
  - `--cov-report=term-missing`: Prints a coverage summary to the terminal, including specific line numbers that lack coverage.
  - `--cov-report=html`: Generates a detailed HTML coverage report (saved to an `htmlcov/` directory by default).

## make.py Integration
Modify `make.py` to support a `test` command.
- When the user runs `python make.py test`, it will execute `pytest` via the `subprocess` module.
- This provides a unified entry point for common project tasks (run, build, test).

## .gitignore Updates
Ensure that coverage artifacts are ignored by Git.
- Add `.coverage` (data file generated by coverage.py).
- Add `htmlcov/` (directory containing the HTML report).
- Add `.pytest_cache/` (cache directory used by pytest).
