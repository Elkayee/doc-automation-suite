## 2026-05-13 - Tkinter Listbox Empty States
**Learning:** In Tkinter, Listbox widgets lack a native concept of "empty states". A common a11y/UX pattern is to insert a placeholder string ("Chua co du an nao") and visually distinguish it (e.g. `listbox.itemconfig(0, foreground='gray')`). However, when doing so, *every* event handler bound to selection or double-click must be updated to explicitly check the foreground color (`if listbox.itemcget(idx, 'foreground') == 'gray'`) and return early. Failing to do so causes the app to attempt to load or execute the placeholder text as if it were valid user data.
**Action:** When implementing an empty state for a Listbox, always audit and guard all associated event bindings (`<<ListboxSelect>>`, `<Double-1>`, deletion buttons, open buttons) to safely ignore the placeholder item.
