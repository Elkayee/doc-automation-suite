## 2025-05-12 - [Performance] Bounded split and fast-path text parsing
**Learning:** In string processing routines such as checking lines of text for markdown fences, an unbounded `str.split('\n')` or `str.replace('\r\n', '\n').replace('\r', '\n').split('\n')` combined with replacing string characters on every run will severely hinder performance by performing string copies and creating lists of memory references to strings.
**Action:** Always optimize loops that search through or build on a subset of string text with max-limits (`split('\n', safe_limit)`) combined with an initial fast path validation check (i.e. returning false if an essential search term doesn't even exist using `in` syntax). Also wrap replaces with `in` conditions so that replace doesn't have to scan through when the search value doesn't exist.
