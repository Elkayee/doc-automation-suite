## 2024-05-08 - Fast-path text optimization over splitlines()
**Learning:** Python's `str.splitlines()` is highly optimized, but substituting `text.replace('\r\n', '\n').replace('\r', '\n').split('\n')` with `text.splitlines()` can subtly break parsing due to how `splitlines()` drops terminal newlines and splits on exotic whitespace (like `\v` or `\f`). Furthermore, attempting to parse iteratively in Python (using loops with `.find()`) on large buffers is slower than C-optimized string operations.
**Action:** When optimizing full-buffer text parsing like markdown block detection, keep the exact `replace().split()` logic to preserve whitespace correctness, but guard it with a fast-path substring check (e.g., `if '```' not in text: return False`). This simple O(1) check bypasses the entire expensive parsing logic in the vast majority of cases (when the token doesn't exist) without breaking correctness.
